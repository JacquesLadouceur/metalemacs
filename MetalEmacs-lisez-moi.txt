INSTRUCTIONS D'INSTALLATION
═══════════════════════════════════════════════════════════════

Windows
───────
1. Dans le terminal: Setx HOME %USERPROFILE%
2. Installer Emacs
3. Copier le dossier .emacs.d dans : C:\Users\VotreNom\
4. Lancer Emacs
5. Patienter 2-3 minutes au premier démarrage
   MetalEmacs va automatiquement :
   - Télécharger et installer MinGit (si Git absent)
   - Compiler les packages Emacs (.elc)

   NOTE : Une fenêtre PowerShell peut apparaître brièvement
   pendant l'installation de Git. C'est normal.

   NOTE : Si votre nom d'utilisateur Windows contient des
   caractères spéciaux (accents, espaces), suivez les
   instructions affichées par MetalEmacs.

macOS
─────
1. Installer Emacs
2. Copier le dossier .emacs.d dans votre dossier personnel (ex. /Users/VotreNom/)
3. Lancer Emacs
4. Patienter ~30 secondes au premier démarrage
   MetalEmacs va automatiquement :
   - Installer Xcode Command Line Tools (si absent)
   - Compiler les packages Emacs (.elc)

   NOTE : Sur macOS, une fenêtre système peut s'ouvrir pour
   l'installation des Command Line Tools. Cliquez « Installer »
   puis redémarrez Emacs une fois terminé.

Linux / Chromebook
──────────────────
Prérequis :
   Sur Chromebook, activez d'abord l'environnement Linux :
   Paramètres → Avancés → Développeurs → Activer Linux

1. Installer les dépendances système :
   sudo apt update && sudo apt upgrade -y
   sudo apt install -y emacs git curl build-essential
2. Copier le dossier .emacs.d dans : ~/
3. Lancer Emacs
4. Patienter quelques minutes au premier démarrage
   (compilation des packages)


═══════════════════════════════════════════════════════════════

PREMIER DÉMARRAGE
═══════════════════════════════════════════════════════════════

Au premier lancement, Emacs va automatiquement :
✓ Détecter votre système d'exploitation
✓ Installer Git si nécessaire (Windows : MinGit, macOS : Xcode CLI)
✓ Compiler les packages (.elc)
✓ Installer les dépendances manquantes
✓ Configurer l'environnement

NE PAS PANIQUER si Emacs affiche des messages pendant
quelques minutes — c'est normal !

MetalEmacs inclut un assistant graphique pour installer
les outils requis pour les cours. L'assistant est
accessible directement depuis le tableau de bord.


═══════════════════════════════════════════════════════════════

MISE À JOUR DE MetalEmacs
═══════════════════════════════════════════════════════════════

Pour mettre à jour MetalEmacs à partir d'une nouvelle
distribution :

1. Télécharger la nouvelle distribution
   (fichier .emacs.d.zip ou dossier emacs)

2. Dans Treemacs, naviguer jusqu'au fichier .emacs.d.zip
   ou jusqu'au dossier emacs contenant .emacs.d

3. Placer le curseur sur le fichier ou le dossier

4. Appuyer sur M (Shift+M) pour lancer la mise à jour

MetalEmacs va automatiquement :
✓ Créer une copie de sauvegarde de votre .emacs.d actuel
  (nommée .emacs.d.backup-YYYYMMDD-HHMMSS)
✓ Extraire ou copier le nouveau .emacs.d
✓ Remplacer votre configuration
✓ Proposer de redémarrer Emacs

PROBLÈMES COURANTS
═══════════════════════════════════════════════════════════════

"Git non trouvé" (Windows)
→ Patienter, MinGit se télécharge automatiquement via PowerShell
→ Si le problème persiste, fermer et relancer Emacs

"Caractères spéciaux dans le nom d'utilisateur" (Windows)
→ Suivre les instructions affichées par MetalEmacs

"Package xxx failed to install"
→ Fermer Emacs et supprimer : .emacs.d/straight/build/
→ Relancer Emacs

"xelatex not found" (Windows)
→ Installer MiKTeX via l'assistant depuis le tableau de bord
→ MiKTeX installe automatiquement les paquets LaTeX manquants

"SWI-Prolog FATAL ERROR: Could not find system resources"
→ Relancer Emacs après l'installation de SWI-Prolog
→ L'assistant configure automatiquement le PATH

"Command Line Tools" (macOS)
→ Une fenêtre système s'ouvre pour l'installation
→ Suivre les instructions puis redémarrer Emacs
